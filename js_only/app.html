The Data Extraction

The process of read and save the Info & Sequence data from pre-processed Fasta-formated text. The Fasta text should be multiple-sequence-aligned, thus the sequence is aligned for analysis. The Fasta text should has a specialized Description line(the line starts with '>'). The line should contain the extra categorical information like 'year', 'host', 'Type', split by Splitters like '|'. A sample Description line : '>A123456|Human|Shanghai|2013|N9'. For this tool to extract the Extra Info, or select some of it, the Splitter and the Mapper is needed:

Splitter - input the splitter character, like '|'.
Mapper - indicate what info is needed and what place is it placed. Suppose the splitter is '|', the Mapper '|1||2|0' means 3 extra-info-data are needed. For a Description line '>A123456|Human|Shanghai|2013|N9', this mapper will get information 'Human', '2013', 'N9', and place them in order as 'N9', 'Human', '2013'. So the Number in the Mapper is the place Order, the place of the Number is the place of the Info-data relative to the splitter. You can use '-' with order to extract the Last number(mostly year), like '|1||-2|0' for '>A123456|Human|Shanghai|1/12/2013|N9'.
Compress - used to remove the redundant 'space[ ]', 'underscore[_]', 'tab[\t]' on each side of extracted info.

After all is set: click 'Extract'!

The Data Picking

The process of picking needed Info & Sequence data from Extracted Data. The extracted data will be displayed in the table above. Clicking the '+' button will open the window of options for picking data. To pick some portion from the extracted data and use them in the next step, there are 4 type of data modification methods:

Range - for column selection.
Use query string like '2,5,10-40,69' to pick column numbered 2, 5, 10, 11, 12 ... 39, 40, 69

Mix - for mixing two or more columns to create a new column.
Use query string like '0|2|3,0|2,2|3' to mix column numbered [0 + 2 + 3], [0 + 2], [2 + 3] Mixed new column will be created by linking the string in each row, like 'a|1|human' from 'a', '1', 'human' of columns 'site52', 'Type', 'Host', maybe. Thus keeping the Enumerate characteristic from each column. If you don't understand this explanation, just try it.

Merge - for the need of combine/merge different value.
Use query string like '0>201X>2011|2013,4>avian>avian|Avian|A' to merge value '2011 2013' to '201X' in column numbered 0, and 'avian Avian A' to 'avian' in column 4

Specified - for row selection, by specifying values.
Use query string like '291=a|b,300=j' to specify rows matching value 'a b' in column numbered 291, and 'c' in column 300. Only the rows matching All specified values will be picked.

-column number starts from 0

The Data Analysis

The process of choosing the analysis method for Info & Sequence data, and check the picked data. The picked data will be displayed in the table above. There are 2 types of methods for data analysis:

Table - for quickly scan the data and find interesting relations between columns or mixed columns.
A table is created by categorizing the data from two columns, each from ExtraInfo and Sequence. The table contains information about the non-randomness distribution: the difference between the observed data distribution and the random hypothesis. The difference is calculated as chi-square-distribution, and the result possibility 'p' as score.

Divide & Slice - for carefully observation of each column of data.
The Divide & Slice method will divide each column into slices by it's enumeration(values inside). Each Slice has it's own binary 'ID array'. Some of the ID array may be the same, or mostly the same. Thus by comparing the slices, we can find relations of 'one value at one column' to 'another value in another column'.
There can be 2 'sub-methods', depending on what counts inside a slice comparison:
-- Quantity -- How many -- The number of the 'relation-pair' is considered as additional information, there can be a matchness value calculated from the number.
-- Quality -- If there has -- The number of the 'relation-pair' is not reliable, the match will be strictly yes/no, but can has a larger size of mix for the simplicity of the determination.
The 'sub-methods' target different data size: Quantity for larger and more average-grouped data; Quality for small size data or a quick analysis for large data.
